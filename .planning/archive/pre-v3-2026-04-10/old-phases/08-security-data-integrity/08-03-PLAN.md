<task type="auto">
  <name>Fix SQL injection in admin product search</name>
  <files>src/app/admin/products/page.tsx</files>
  <action>
Current code (line 40) passes user input directly to `.ilike()`:

```typescript
query = query.ilike('name', `%${searchQuery}%`);
```

This is vulnerable to SQL injection via special characters in PostgREST query syntax.

**Fix:**

1. **Import escapePostgrestQuery** from product-service (add to imports at top):
   ```typescript
   import { escapePostgrestQuery } from '@/lib/supabase/product-service';
   ```

2. **Escape search query before using** (modify line 40):
   ```typescript
   const escapedQuery = escapePostgrestQuery(searchQuery);
   query = query.ilike('name', `%${escapedQuery}%`);
   ```

The `escapePostgrestQuery` function is already implemented in product-service.ts (used by public search on homepage). It escapes special characters that have meaning in PostgREST pattern matching: `%`, `_`, `\`, `*`, `.`, `[`, `]`, `(`, `)`.

Reference existing usage pattern from public search.
  </action>
  <verify>
Check import exists: `grep "escapePostgrestQuery" src/app/admin/products/page.tsx`
Check usage: `grep "escapedQuery" src/app/admin/products/page.tsx`
TypeScript compiles: `npx tsc --noEmit`
  </verify>
  <done>
Admin product search escapes user input before database query.
SQL injection via search parameter is blocked.
  </done>
</task>

<task type="auto">
  <name>Fix open redirect in admin login</name>
  <files>src/app/admin/login/page.tsx</files>
  <action>
Current code (line 15) trusts redirect parameter from URL:

```typescript
const redirect = searchParams.get('redirect') || '/admin';
```

Then uses it directly (line 36):

```typescript
router.push(redirect);
```

This allows open redirect attack via crafted URL: `/admin/login?redirect=https://evil.com`

**Fix:**

1. **Add redirect validation helper** (add after imports, before LoginForm component):
   ```typescript
   function isValidRedirect(url: string): boolean {
     // Only allow relative URLs starting with /
     // Block absolute URLs, protocol-relative URLs, and javascript: URIs
     return url.startsWith('/') && !url.startsWith('//');
   }
   ```

2. **Validate redirect parameter** (replace line 15):
   ```typescript
   const redirectParam = searchParams.get('redirect') || '/admin';
   const redirect = isValidRedirect(redirectParam) ? redirectParam : '/admin';
   ```

This ensures redirect always points to same-origin path, never external site.
  </action>
  <verify>
Check helper function exists: `grep "isValidRedirect" src/app/admin/login/page.tsx`
Check validation usage: `grep "isValidRedirect(redirectParam)" src/app/admin/login/page.tsx`
TypeScript compiles: `npx tsc --noEmit`
Test case: URL `/admin/login?redirect=https://evil.com` should redirect to `/admin` (not evil.com)
  </verify>
  <done>
Admin login validates redirect parameter.
Open redirect vulnerability is blocked.
Only same-origin paths allowed.
  </done>
