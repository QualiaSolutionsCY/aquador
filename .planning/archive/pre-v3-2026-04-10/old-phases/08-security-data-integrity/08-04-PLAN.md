<task type="auto">
  <name>Tighten CSP directives</name>
  <files>next.config.mjs</files>
  <action>
Update Content-Security-Policy header (line 62):

**Current CSP issues:**
1. `script-src` includes `'unsafe-eval'` — dangerous, allows eval() and Function() constructor
2. `media-src` uses wildcard `https:` — allows any HTTPS media source

**Changes:**

1. **Remove unsafe-eval** from script-src:
   ```javascript
   script-src 'self' 'unsafe-inline' https://vercel.live https://*.vercel.live https://js.stripe.com https://*.sentry.io;
   ```
   (Remove `'unsafe-eval'` from current directive)

2. **Restrict media-src** to specific domains:
   ```javascript
   media-src 'self' https://static1.squarespace.com https://images.squarespace-cdn.com https://*.supabase.co;
   ```
   (Replace `media-src https://static1.squarespace.com https:;` with specific allowlist)

**Note on unsafe-inline:**
SEC-06 requirement says "restrict unsafe-inline WHERE POSSIBLE." Cannot remove `'unsafe-inline'` from script-src or style-src yet — Next.js and Tailwind require inline scripts/styles. Would need nonce-based CSP for that (major refactor, out of scope for v1.1 milestone). Leaving unsafe-inline is acceptable per requirement wording. Removing unsafe-eval fully addresses the critical risk.

**Full updated CSP** (line 62, replace entire value):
```javascript
value: "default-src 'self'; img-src 'self' https: data: blob:; script-src 'self' 'unsafe-inline' https://vercel.live https://*.vercel.live https://js.stripe.com https://*.sentry.io; style-src 'self' 'unsafe-inline' https://fonts.googleapis.com; font-src 'self' https://fonts.gstatic.com; frame-src https://www.google.com https://js.stripe.com https://vercel.live; media-src 'self' https://static1.squarespace.com https://images.squarespace-cdn.com https://*.supabase.co; connect-src 'self' https://api.stripe.com https://vercel.live https://*.vercel.app wss://ws-us3.pusher.com https://*.sentry.io https://*.supabase.co wss://*.supabase.co;",
```

Key changes: removed `'unsafe-eval'`, tightened `media-src`. unsafe-inline remains as documented limitation.
  </action>
