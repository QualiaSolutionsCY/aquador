Create `src/lib/analytics/performance-monitor.ts` with performance tracking helpers.

**Functions to implement:**

1. `usePerformanceMonitor(componentName)` - Hook to monitor FPS and frame drops
   - Uses requestAnimationFrame to measure frame times
   - Calculates rolling average FPS over 60 frames
   - Detects frame drops (frame time >33ms = below 30fps)
   - Tracks performance metrics on unmount if degradation detected (>10 frame drops)
   - Metadata: componentName, averageFPS, frameDropCount, deviceType, timestamp

2. `trackAnimationPerformance(componentName, frameDropCount, avgFPS)` - Track animation performance issues
   - `componentName`: identifier for which component had the issue (e.g., 'animation_budget_global', '3d_rotation')
   - `frameDropCount`: number of dropped frames detected
   - `avgFPS`: average FPS measured during the monitoring window
   - Only tracks if avgFPS <45 (performance issue) or frameDropCount >10 (significant drops)
   - This function is called internally by `usePerformanceMonitor` on unmount

3. `trackCinematicEngagement(elementType, action, duration)` - Track cinematic element interactions
   - ElementType: 'page_transition' | 'parallax_section' | 'filter_animation' | 'category_transition'
   - Action: 'view' | 'interact' | 'complete'
   - Duration: milliseconds (for 'complete' action)
   - Measures effectiveness of cinematic enhancements

4. `measureFPS(callback)` - Helper to measure FPS over time period
   - Uses requestAnimationFrame loop
   - Returns average FPS and frame count
   - Used internally by usePerformanceMonitor

**Implementation notes:**
- Import `track` from '@vercel/analytics'
- Import `getDeviceType` from '@/lib/analytics/engagement-tracker'
- Use requestAnimationFrame (RAF) for accurate frame timing
- Clean up RAF loop on unmount
- Only track significant performance issues (not every render)
- Add performance.now() for high-precision timing

**Performance pattern:**
```typescript
export function usePerformanceMonitor(componentName: string) {
  const frameTimesRef = useRef<number[]>([]);
  const rafIdRef = useRef<number>();
  const lastFrameTimeRef = useRef<number>(performance.now());

  useEffect(() => {
    let frameDropCount = 0;

    const measureFrame = (currentTime: number) => {
      const frameTime = currentTime - lastFrameTimeRef.current;
      lastFrameTimeRef.current = currentTime;

      if (frameTime > 33) { // >33ms = <30fps = frame drop
        frameDropCount++;
      }

      frameTimesRef.current.push(frameTime);
      if (frameTimesRef.current.length > 60) {
        frameTimesRef.current.shift(); // Keep last 60 frames
      }

      rafIdRef.current = requestAnimationFrame(measureFrame);
    };

    rafIdRef.current = requestAnimationFrame(measureFrame);

    return () => {
      if (rafIdRef.current) {
        cancelAnimationFrame(rafIdRef.current);
      }

      // Track on unmount if performance issues detected
      if (frameDropCount > 10) {
        const avgFPS = 1000 / (frameTimesRef.current.reduce((a, b) => a + b, 0) / frameTimesRef.current.length);
        trackAnimationPerformance(componentName, frameDropCount, avgFPS);
      }
    };
  }, [componentName]);
}
```

Note: All three refs (`frameTimesRef`, `rafIdRef`, `lastFrameTimeRef`) are declared at component top level — the pattern above is correct. Do NOT move any `useRef` call inside the `useEffect` body.
