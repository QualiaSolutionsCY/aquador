  <action>
Create `src/lib/analytics/product-engagement.ts` with product-specific tracking helpers.

**Functions to implement:**

1. `useProductViewTracking(productSlug, productName)` - Hook to track time on product page
   - Records entry timestamp on mount
   - Tracks 'product_view_time' event on unmount with duration in seconds
   - Only tracks if duration > 3 seconds (meaningful engagement, not bounces)
   - Metadata: productSlug, productName, duration, deviceType

2. `trackFilterChange(filterType, value, resultCount)` - Track filter interactions
   - FilterType: 'category' | 'type' | 'search'
   - Value: selected filter value (e.g., 'women', 'perfume')
   - ResultCount: number of products after filter applied
   - Enables funnel analysis (which filter combinations lead to purchases)

3. `trackCategoryTransition(fromCategory, toCategory, method)` - Track category navigation
   - FromCategory: previous category slug (or 'shop' for main page)
   - ToCategory: new category slug
   - Method: 'click' | 'filter' | 'direct' (how user navigated)
   - Maps discovery flow patterns

4. `trackQuickView(productSlug, productName, duration)` - Track quick view interactions
   - ProductSlug: product identifier
   - ProductName: for human-readable reports
   - Duration: milliseconds modal/quick view was open
   - Distinguishes browsing behavior from deep product engagement

- Import `getDeviceType` from '@/lib/analytics/engagement-tracker'

  <action>
The product detail page at `src/app/products/[slug]/page.tsx` is a Server Component (no 'use client' directive). React hooks cannot be called in Server Components. This requires a dedicated client component wrapper.

**Step 1 — Create `src/components/products/ProductViewTracker.tsx`:**

```typescript
'use client';

import { useProductViewTracking } from '@/lib/analytics/product-engagement';

interface ProductViewTrackerProps {
  productSlug: string;
  productName: string;
}

export function ProductViewTracker({ productSlug, productName }: ProductViewTrackerProps) {
  useProductViewTracking(productSlug, productName);
  return null; // Renders nothing, only side effect
}
```

This component is a zero-render tracking side-effect: it mounts when the page loads, starts the timer, and fires the analytics event on unmount (when user navigates away).

**Step 2 — Update `src/app/products/[slug]/page.tsx`:**

1. Import `ProductViewTracker` from '@/components/products/ProductViewTracker'
2. Read the file first to find the correct location — render `<ProductViewTracker />` inside the page's JSX, after product data is available:
   ```typescript

   ```
3. Place it near the top of the return, alongside other non-visual components

**Important:**
- Do NOT add 'use client' to the page — keep it as a Server Component for SEO/SSR
- Do NOT call useProductViewTracking directly in the page
