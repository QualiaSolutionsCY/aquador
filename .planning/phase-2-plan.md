# Phase 2: Stack & Type Unification

**Goal:** When this phase ships, the project runs on Next 16 + React 19 + ESLint 9 with `npm run build`, `npm run lint`, `npm test`, and `npm run test:e2e -- --project=chromium` all green; `LegacyProduct` is fully deleted; every consumer of product data reads the snake_case Supabase shape; and the 24 pre-existing Jest failures are repaired (not skipped).

**Why this phase:** Phase 1 shipped tokens on a stack two majors behind. Phase 3 builds primitive components — those primitives must be authored against React 19 + Next 16 idioms (Server Components, async APIs, React Compiler-ready) and against a single Product type. Doing the stack/type cleanup now means Phase 3 builds on solid substrate; doing it after Phase 3 forces a second rewrite. The Jest red baseline is fixed as part of this phase because a green test suite is the only way verifier checks for Phase 3 mean anything — a baseline of "24 known failures" is indistinguishable from "26 known failures".
